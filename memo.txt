# うまさば-odds サービス定義書（System Specification）

## 1. サービスコンセプト
本サービスは、競馬シミュレーションやコミュニティイベントにおける「事前の盛り上がり」を最大化するための、リアルタイム・オッズ算出プラットフォームです。
実際の着順を競うのではなく、コミュニティ内の「期待値」や「熱量」をオッズとして可視化し、ユーザー同士の交流を促進することを目的としています。

## 2. コア・メカニズム
- **人気投票型オッズ:** 3連複の組み合わせに対する投票数からリアルタイムに倍率を算出。
- **ベッティングフリー:** 金銭やポイントではなく、純粋な「応援・予想」をベースとしたアンケート形式。
- **リアルタイム同期:** Firebaseを活用し、リロードなしでチャートやオッズが最新状態に更新。

## 3. ページ別機能解説

### 3.1 投票ページ (`index.html`)
- **役割:** サービスの玄関口であり、投票を行うメイン画面。
- **主な機能:**
    - **複数レース対応:** 開催中のレース一覧から選択して投票が可能。
    - **直感的な操作:** 3頭の馬を選択し、コメントを添えて即座に投票完了。
    - **重複チェック:** 同一馬の選択防止などのバリデーション機能。

### 3.2 オッズ詳細ページ (`odds.html`)
- **役割:** 投票状況の可視化と、コミュニティの反応を分析する画面。
- **主な機能:**
    - **データビジュアライズ:** Chart.jsによる得票率の円グラフ表示。
    - **コメントビューワー:** 投票時に寄せられた「推しポイント」を一覧表示。
    - **ユーザー連携:** コメント投稿者の名前から、その人の「個人ページ（user.html）」へ直接遷移可能。
    - **クイック切替:** 他のレース結果へスムーズに移動できるプルダウンメニュー。

### 3.3 ユーザーページ (`user.html`)
- **役割:** 参加者個人のアイデンティティと活動記録を表示する画面。
- **主な機能:**
    - **パーソナライズ表示:** `settings.html` で設定したアイコン画像、一言コメント、所属サークル、推し馬情報を表示。
    - **ダイナミック・テーマ:** 設定された「推し馬」に応じたパーソナルカラーがページ全体に適用（背景やアクセントカラーが変化）。
    - **投票履歴:** そのユーザーが過去にどのレースに投票したかのタイムライン。

### 3.4 設定ページ (`settings.html`)
- **役割:** ユーザー自身の情報をカスタマイズする画面。
- **主な機能:**
    - **アイコンアップロード:** 独自の画像をアップロードし、切り抜き（クロップ）して設定可能。
    - **プロフィール編集:** ユーザー名、サークル名、一言コメントの更新。
    - **推し馬セレクト:** 自分のテーマカラーを決定する「推し馬」の選択。

## 4. UI/UX デザイン指針
- **アダプティブ・レイアウト:** PCでは2カラム構成（グラフとリストを並列）、スマホでは縦積みの1カラム構成へ自動最適化。
- **ダークモード連動:** OS設定やユーザーの好みに合わせた外観切り替え。
- **ビジュアル・フィードバック:** ページ遷移時のフェードイン効果や、テーマカラーの動的変更による高級感の演出。

## 5. 技術スタック（概要）
- **Frontend:** HTML5, SCSS (BEM), JavaScript (ES6+)
- **Backend:** Firebase Realtime Database
- **Storage:** Firebase Storage (アイコン画像管理)
- **Infrastructure:** Vercel
